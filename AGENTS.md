# Правила для `hookah-api`

## Роль репозитория

Backend-слой с бизнес-логикой, валидацией, API-контрактом и интеграцией с PostgreSQL.

## Стек

- NestJS
- TypeScript strict mode
- REST API
- Swagger / OpenAPI
- class-validator
- Jest
- ESLint
- Prettier

## Локальные правила

- Каждый новый модуль должен отражать доменный контекст, а не техническую папку ради папки.
- DTO описываем так, чтобы из них было удобно строить OpenAPI-контракт.
- Глобальные пайпы, префиксы и versioning настраиваем централизованно.
- Не добавляем `any`, если контракт можно выразить типом или DTO.

## Коммиты

- Пишем на русском языке.
- В одном коммите не смешиваем инфраструктурную настройку и доменную логику без причины.
- Перед коммитом запускаем `npm run check`.
